### Phase 5 PostFX Color Assignments

Every glow color is sourced from the Void Ethereal palette established in Phase 1 — no warm yellows, no electric cyans, no traffic-light tints (D-03, specifics).

| Element | Base Glow | Upgraded Glow | Source |
|---------|-----------|---------------|--------|
| Blaster turret | `0xeef2ff` (accent white) | `0xeef2ff` (accent white — no color shift; upgrade signal remains alpha pulse + `0xffdd44` tint) | D-03, D-04 |
| Zapper turret | `0x9966ff` (light violet) | `0xeef2ff` (accent white) | D-03, D-04 |
| Slowfield turret | `0x6a4c93` (deep nebula purple — THEME.nebula[2]) | `0xeef2ff` (accent white) | D-03, D-04 |
| Wall | none — walls do not receive glow | n/a | D-02 |
| Command core | `0xeef2ff` (accent white) | n/a — core cannot be upgraded | D-03 |
| Vignette | pure black (`0x000000`, Phaser addVignette default) | n/a | D-08 |

### Glow Target Scope (D-02 — strict)

| Class | Receives Glow? |
|-------|----------------|
| Turrets (blaster, zapper, slowfield) | yes |
| Walls | no |
| Command core | yes |
| Bugs (swarmer, brute, spitter, boss) | no |
| Bullets (turret + spitter) | no |
| Particles (death, muzzle, build, shockwave, slowfield aura, zapper trail) | no |
| Menu text / HUD text | no |

Rationale: "Defenders glow, threats don't" (D-02, specifics). Per-frame FX cost bounded by `turrets.length + 1`.

## PostFX Contract

### Performance Budget

Hard constraint: maintain 60fps with 60 bugs + 70 bullets + particles active, plus glow on `turrets.length + 1` sprites and one vignette postFX controller on the main camera. Vignette tween runs at most once per phase transition (~every 20s in build, variable in wave) — negligible cost.

### Renderer Detection & Canvas Fallback

| Property | Value | Source |
|----------|-------|--------|
| Detection call | `const isWebGL = this.game.renderer.type === Phaser.WEBGL;` | D-12 |
| Where | Inline check in each postFX-enabled scene's `create()` — no shared util, no registry stash | D-12 |
| Scopes using detection | `GameScene.create()`, `MainMenuScene.create()`, `GameOverScene.create()`, `Turret` constructor, `Turret.upgrade()`, `Turret.destroy()` | D-10, D-11, D-14 |
| On Canvas: glow | Skipped entirely — no `sprite.preFX.addGlow(...)` call is made. No fallback halo texture. | D-10 |
| On Canvas: vignette | Skipped entirely — no `camera.postFX.addVignette(...)` call is made. No Graphics overlay fallback. | D-11 |
| On Canvas: warning | `console.warn('[postfx] Canvas renderer detected — glow disabled')` logged once per scene boot (one line total per scene, not per turret). | D-10 |

Canvas users get a fully playable game without the atmospheric polish — no extra code paths, no fallback visuals.

### Per-Sprite Glow (THEME-04)

Applied via `sprite.preFX.addGlow(color, outerStrength, innerStrength)` directly on the target Phaser sprite. No camera-level bloom, no custom shader pipeline (D-01).

| Property | Value | Source |
|----------|-------|--------|
| API | `Phaser.GameObjects.Components.FX.addGlow(color, outerStrength, innerStrength, knockout, quality, distance)` — first three args only; rest left at Phaser defaults | D-01 |
| Attach point (turret) | `turret.sprite.preFX` — NEVER `turret` itself (Turret is a composite class wrapping a sprite) | code_context, D-01 |
| Attach point (core) | `this.coreSprite.preFX` in `GameScene.renderCore()` | code_context |
| Timing (turret) | Applied in `Turret` constructor immediately after `this.sprite = scene.add.sprite(...)` | integration notes |
| Timing (core) | Applied in `GameScene.renderCore()` after the existing breathing tween is set up | integration notes |
| Reactivity | **Constant ambient** — set once, never tweened, never event-reactive (no fire/damage flashes) | D-05 |
| Upgrade path | `this.sprite.preFX.clear(); this.sprite.preFX.addGlow(upgradedColor, ...)` inside `Turret.upgrade()` | integration notes |
| Starting outerStrength | 2 (tune during execution — keep 60fps budget) | discretion |
| Starting innerStrength | 1 (tune during execution) | discretion |
| Core outerStrength | 3 (core is larger, gets slightly more outer glow) | discretion |
| Core innerStrength | 1 | discretion |

Glow is intentionally calm — it pairs with the Phase 2 idle alpha pulse (which already provides "aliveness") and avoids visual noise on top of muzzle flash, damage flash tint, and Phase 3/4 particle effects (D-05).

### Vignette (THEME-05)

Applied via `camera.postFX.addVignette(x, y, radius, strength)` on the target scene's main camera (D-06). Phaser 3.80+ API.

| Property | Value | Source |
|----------|-------|--------|
| API | `Phaser.Cameras.Scene2D.Camera.postFX.addVignette(x, y, radius, strength)` | D-06 |
| Color | pure black (Phaser default — not a parameter of addVignette) | D-08 |
| x (center) | 0.5 (canvas center, normalized) | D-13, discretion |
| y (center) | 0.5 (canvas center, normalized) | D-13, discretion |
| radius | 0.5 (tune during execution for the 7×7 grid framing at 1920x1080) | D-13, discretion |
| Build-phase strength | 0.25 | D-09 |
| Wave-phase strength | 0.40 | D-09 |
| Intensity philosophy | subtle atmospheric framing — "felt, not seen" — if player consciously notices the tween, it's too strong | D-07, specifics |

### Phase-Reactive Vignette Tween (GameScene only)

Wave phase subtly leans in; build phase exhales (D-09, specifics).

| Property | Value | Source |
|----------|-------|--------|
| Trigger event | `phase-changed` emitted on `GameScene.events` with `{ phase: 'build' \| 'wave' }` | code_context |
| Emission sites | `GameScene.js:156` (startBuildPhase), `GameScene.js:184` (startWavePhase), `GameScene.js:201` (onWaveComplete→build) | code_context |
| Tween target | Vignette FX controller's `strength` property | D-09 |
| Tween from | current `strength` (whichever phase was previously active) | tween semantics |
| Tween to | `POSTFX.VIGNETTE.buildStrength` (0.25) on `build`, `POSTFX.VIGNETTE.waveStrength` (0.40) on `wave` | D-09 |
| Duration | 600ms | D-13 |
| Easing | `Sine.easeInOut` | discretion (matches existing Phase 2/3 tweens) |
| Cleanup | Tween reference stored as `this._vignetteTween`; destroyed in existing `events.once('shutdown', ...)` handler along with `this.cameras.main.postFX.clear()` | D-09, Phase 1 D-03 tween-safety |
| Initial strength on scene boot | `POSTFX.VIGNETTE.buildStrength` (GameScene starts in build phase per `init()`) | code_context |

### Scene Scope (D-14)

| Scene | Glow | Vignette | Notes |
|-------|------|----------|-------|
| GameScene | applied to each turret sprite + coreSprite | applied to `cameras.main`; phase-reactive tween attached | the only scene with both effects and the only scene with the phase tween |
| MainMenuScene | n/a — no turrets or core in this scene, so D-14 is vacuously satisfied (do NOT invent menu-text glows to force D-14 literally) | applied to `cameras.main`; **static strength = `POSTFX.VIGNETTE.buildStrength` (0.25)** | menu-calm static vignette only |
| GameOverScene | n/a — no turrets or core in this scene | applied to `cameras.main`; **static strength = `POSTFX.VIGNETTE.buildStrength` (0.25)** | static vignette only |
| UIScene | **exempt** — never apply postFX here | **exempt** — HUD must remain unshaken and un-vignetted for readability | matches SHAKE-04 isolation pattern from Phase 4 D-10 |

### Cleanup Patterns

All persistent FX handles and tweens MUST be cleaned up on scene/entity teardown to honor the Phase 1 D-03 tween-safety pattern.

| Site | Cleanup action |
|------|----------------|
| `Turret.destroy()` | `this.sprite.preFX?.clear();` before `this.sprite.destroy();` |
| `Turret.upgrade()` (re-apply path) | `this.sprite.preFX.clear();` then re-call `preFX.addGlow(upgradedColor, ...)` — also guarded by `isWebGL` check |
| `GameScene` `events.once('shutdown', ...)` block (lines 92–99) | add `this._vignetteTween?.destroy();` and `this.cameras.main.postFX.clear();` |
| `MainMenuScene` / `GameOverScene` | no explicit cleanup required — scene teardown disposes the camera and its postFX controllers automatically (no tweens to kill since vignette is static) |

### Config Structure (D-13)

Add a new top-level `POSTFX` frozen export to `src/config/GameConfig.js`, sibling to `THEME` and `VFX`. Every nested object MUST be frozen (project convention per `AGENTS.md` and the rest of `GameConfig.js`).

```js
export const POSTFX = Object.freeze({
  GLOW: Object.freeze({
    blaster:   Object.freeze({ base: 0xeef2ff, upgraded: 0xeef2ff, outerStrength: 2, innerStrength: 1 }),
    zapper:    Object.freeze({ base: 0x9966ff, upgraded: 0xeef2ff, outerStrength: 2, innerStrength: 1 }),
    slowfield: Object.freeze({ base: 0x6a4c93, upgraded: 0xeef2ff, outerStrength: 2, innerStrength: 1 }),
    core:      Object.freeze({ color: 0xeef2ff,                    outerStrength: 3, innerStrength: 1 }),
  }),
  VIGNETTE: Object.freeze({
    x: 0.5,
    y: 0.5,
    radius: 0.5,
    buildStrength: 0.25,
    waveStrength: 0.40,
    transitionDuration: 600,
    transitionEase: 'Sine.easeInOut',
  }),
});
```

Notes:
- No `walls` entry under GLOW — walls do not receive glow (D-02). Turret constructor must branch on `type === 'wall'` and skip `addGlow` entirely.
- Color values are **numeric hex literals** (`0xeef2ff`), NOT CSS strings (`'#eef2ff'`). `preFX.addGlow` expects numeric. The existing `THEME.accent` is stored as `'#eef2ff'`; POSTFX.GLOW restates the same color in numeric form rather than converting at runtime (code_context).
- Exact `outerStrength`, `innerStrength`, and vignette `radius` are starting values — tune during execution while keeping the 60fps budget (discretion).
