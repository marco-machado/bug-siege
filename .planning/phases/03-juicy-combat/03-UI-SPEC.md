| `vfx.spitter` | `#ffa34d` | Spitter death burst tint |

| `vfx.shockwave` | `#eef2ff` | Core damage ring + radial sparks (accent-aligned) |

### Accent Drift Note (MUST PRESERVE)

**Contract target:** `THEME.accent = '#eef2ff'` (lavender-white). Confirmed in `src/config/GameConfig.js:139`.

**Known drift:** Some shipped runtime code still uses `#00ff88` (mint) for accent-like UI highlights. This drift is **out of scope for Phase 3** and will be reconciled in Phase 7 (Cohesive Theme, requirement THEME-03). For Phase 3:

- All **new** VFX (placement sparkle, muzzle flash, boss white-flash frame) MUST read from `THEME.accent` resolving to `#eef2ff`.
- Existing `#00ff88` usages in unrelated code MUST NOT be touched.
- Swarmer death burst intentionally uses the literal mint `#00ff88` because it mirrors the existing swarmer sprite color, NOT because it is the accent.

## Phase-Specific VFX Contract

This is the authoritative specification for every particle event in Phase 3. The executor MUST match these values exactly. Values are chosen to hit the upper bound of the project's safe particle budget (juicier feel while preserving 60fps).

### Texture Registry

All textures are generated procedurally in BootScene alongside existing sprites (no external assets). Textures are reused across events to keep the atlas small.

| Texture Key | Size | Shape | Source of Truth | Used By |
|-------------|------|-------|-----------------|---------|
| `particle-soft` | 16×16 | Soft radial dot (white, additive-friendly) | BootScene procedural | Bug death, muzzle flash, core sparks |
| `particle-spark` | 16×16 | 4-point sparkle star | BootScene procedural | Placement sparkle |
| `particle-ring` | 64×64 | Hollow stroked ring (2px stroke) | BootScene procedural | Core shockwave |
| `particle-glow` | 20×20 | Brighter soft dot (higher inner alpha) | BootScene procedural | Boss death inner burst |

## Performance Budget

| Metric | Target | Notes |
|--------|--------|-------|
| On-screen particles (peak) | **<320** | Allows 10% slack over the project's 300 target |
| Frame rate floor | **60fps** | With 60 bugs + 70 bullets + full particle load |
| Simultaneous emitters (worst case) | ~12 | 10 bug deaths + 1 shockwave + 1 muzzle flash burst |
| Particle atlas size (bytes) | <4KB | 4 tiny textures at 16–64px, procedurally generated |
| Bug pool deaths per second (cap) | 15 | 15 × 18 = 270 particles baseline; boss overlap adds 60 |

**Validation:** Executor must verify with `npm run build` and visual QA across a full 10-wave session. If peak particle count exceeds 320 or frame rate dips below 55fps sustained, reduce per-event counts in this order: (1) muzzle flash 8→6, (2) placement sparkle 20→16, (3) regular bug death 18→14.

**Renderer parity:** All contracts MUST render identically on WebGL and Canvas. No renderer branching. Phaser's `ParticleEmitter` supports both; accept the Canvas performance trade-off (typically 20–30% slower but still above 60fps for this budget).
