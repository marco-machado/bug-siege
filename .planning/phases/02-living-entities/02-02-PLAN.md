  <action>
In `renderCore()`, immediately after the `this.coreSprite = this.add.sprite(...).setDisplaySize(...)` line, insert:

```javascript
const baseScale = this.coreSprite.scaleX;
this.tweens.add({
  targets: this.coreSprite,
  scaleX: { from: baseScale, to: baseScale * 1.06 },
  scaleY: { from: baseScale, to: baseScale * 1.06 },
  duration: 1800,
  ease: 'Sine.easeInOut',
  yoyo: true,
  repeat: -1,
});
```

Critical: `baseScale` must be read from `this.coreSprite.scaleX` AFTER `setDisplaySize` is called (it's chained on the same line). `setDisplaySize` internally calculates `scaleX` relative to the texture's natural dimensions. Tweening to absolute `1.06` would be wrong — using `baseScale * 1.06` ensures the pulse is always 6% relative to the actual display size.

The `damageCore()` method in GameScene uses `setTintFill`/`clearTint` which operates on the `tint` property, fully orthogonal to `scaleX`/`scaleY`. No coordination with the tween needed.
  </action>

  <action>
Three insertions into Turret.js:

**1. In the constructor, before the closing `}` brace (after `this.hpBarFill = ...`):**

```javascript
this.idleTween = null;
if (type !== 'wall') {
  this.idleTween = scene.tweens.add({
    targets: this.sprite,
    alpha: { from: 0.75, to: 1.0 },
    duration: 1200,
    ease: 'Sine.easeInOut',
    yoyo: true,
    repeat: -1,
  });
}
```

Walls have no idle glow — they are static barriers, not active turrets. Alpha pulse uses `alpha` not `tint` because `tint` is claimed by the upgrade state (`0xffdd44`).

**2. In upgrade(), before `this.sprite.setTint(0xffdd44);`:**

```javascript
if (this.idleTween) {
  this.idleTween.destroy();
  this.idleTween = this.scene.tweens.add({
    targets: this.sprite,
    alpha: { from: 0.65, to: 1.0 },
    duration: 900,
    ease: 'Sine.easeInOut',
    yoyo: true,
    repeat: -1,
  });
}
```

Use `idleTween.destroy()` (not `.stop()`) matching the `hpTween.destroy()` pattern in `updateHpBar`. Destroying before recreating prevents two simultaneous tweens on the same sprite (Pitfall 5 from RESEARCH.md).

**3. In destroy(), before `this.sprite.destroy();` (first line of destroy()):**

```javascript
if (this.idleTween) {
  this.idleTween.destroy();
  this.idleTween = null;
}
```

Must come before `this.sprite.destroy()` because the tween targets `this.sprite`. Destroying the tween first prevents Phaser from updating a destroyed game object on the next frame.
